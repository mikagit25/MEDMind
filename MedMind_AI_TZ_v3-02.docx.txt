Структура каждого JSON-модуля

1. Общее описание проекта
1.1 Что такое MedMind AI
MedMind AI — глобальная образовательная платформа для врачей, ветеринаров и студентов медицинских специальностей. Платформа объединяет структурированный учебный контент, AI-тьютора на базе Claude API, интервальное повторение флэшкарт и реальные клинические кейсы.


Ключевые принципы платформы:
* AI как мозг — Claude API отвечает на вопросы, объясняет концепции, генерирует новый контент
* Доказательная медицина — весь контент основан на клинических гайдлайнах и PubMed
* Переиспользование контента — базовые дисциплины создаются один раз и применяются во всех специальностях
* Персонализация — адаптация к уровню и прогрессу пользователя
* Ветеринария как отдельный рынок — уникальная особенность, отличающая от конкурентов

2.2 Маршрутизация AI-запросов
Критически важная логика — выбор модели в зависимости от типа запроса и тарифа пользователя:
* Free tier → Ollama (Llama 3 локально) — бесплатно
* Простые вопросы (Student/Pro) → Claude Haiku ($0.00025/1K input) — быстро и дёшево
* Сложные медицинские вопросы → Claude Sonnet ($0.003/1K input) — высокое качество
* Экспертный уровень → Claude Opus — только по запросу


Экономия: маршрутизация Haiku/Sonnet снижает затраты на AI в 3-5 раз по сравнению с использованием только Sonnet.

4. Импорт готовых JSON-модулей — КРИТИЧЕСКИЙ ПРИОРИТЕТ
Это самый важный первый шаг. 70 модулей уже созданы. Они должны быть загружены в БД при первом запуске приложения.


4.1 Структура файлов для импорта
Файлы хранятся в директории /data/modules/ (скопировать из outputs при деплое):
module_BASE-CARDIO-ANATOMY-001.json
module_CARDIO-001.json
... (70 файлов всего)


4.2 Скрипт импорта (scripts/import_modules.py)
Claude Code должен создать и запустить этот скрипт как часть начальной настройки:
* Читает все module_*.json файлы из директории
* Для каждого модуля: создаёт запись в таблице modules с полем content = весь JSON
* Извлекает lessons, flashcards, mcq_questions, clinical_cases в отдельные таблицы
* Генерирует embeddings для каждого урока через Claude/OpenAI API
* Помечает модуль is_published = true после успешного импорта
* Логирует ошибки, не прерывает импорт при единичной ошибке
* Идемпотентен — повторный запуск обновляет существующие, не дублирует

6.2 Система промптов (реализовать все)
Промпты для генерации контента (ContentPrompts)
* generate_module_structure(specialty, topic, level) — структура модуля
* generate_lesson_content(specialty, lesson_title, key_concepts, level) — урок
* generate_flashcards(specialty, topic, count) — флэшкарты
* generate_mcq_questions(specialty, topic, difficulty, count) — MCQ
* generate_clinical_case(specialty, topic, difficulty) — клинический кейс
* generate_branching_case(specialty, topic, difficulty) — интерактивный кейс с ветвлением


Промпты для AI-тьютора (TutorPrompts)
* tutor_system_prompt() — системный промпт (личность, стиль, принципы)
* explain_concept(concept, level) — объяснение концепта
* quiz_mode(topic, previous_mistakes) — устный экзамен
* case_discussion(case_data, user_decision) — обсуждение кейса


Промпты для адаптации (AdaptationPrompts)
* validate_medical_content(content, type) — валидация точности
* adapt_content_difficulty(content, user_level, performance) — адаптация сложности
* adapt_human_content_to_vet(content, species) — адаптация под ветеринарию


Промпты для справочника препаратов (DrugPrompts)
* generate_drug_profile(drug_name, specialty) — полный профиль препарата
* generate_drug_interactions(drug_list) — проверка взаимодействий
* generate_drug_learning_materials(drug_name, exam_type) — учебные материалы


Ветеринарные промпты (VeterinaryPrompts)
* generate_vet_module_structure(specialty, topic, species, level) — ветеринарный модуль

16. Критические требования — не упустить
Эти пункты должны быть реализованы правильно с первого раза. Ошибки здесь критичны.


16.1 Медицинская точность
* HIPAA-дисклеймер в каждом AI-ответе — не диагностический инструмент
* AI не должен давать конкретные дозировки без оговорки о проверке врачом
* Клинические кейсы — образовательные, не для реальных пациентов


16.2 Безопасность
* Никогда не хранить API ключи в коде (только .env)
* Все SQL запросы через parameterized queries (нет конкатенации строк)
* Sanitize все входные данные перед передачей в AI промпты
* Логировать попытки несанкционированного доступа


16.3 Производительность
* Кэшировать AI ответы в Redis — одинаковые вопросы не оплачиваются дважды
* Пагинация во всех списках (по умолчанию limit=20)
* Ленивая загрузка тяжёлого контента (embeddings, изображения)
* Индексы на все поля используемые в WHERE и JOIN


16.4 Ветеринарный режим
* Дозировки для разных видов — РАЗНЫЕ. Не путать кошку и собаку
* Предупреждение о токсичности (например: парацетамол токсичен для кошек)
* Препараты одобренные для людей могут быть запрещены для животных


16.5 Контент
* 70 JSON-модулей ОБЯЗАТЕЛЬНО импортировать при первом запуске
* Не генерировать контент заново — он уже создан и проверен
* При добавлении новых модулей — использовать тот же JSON формат
